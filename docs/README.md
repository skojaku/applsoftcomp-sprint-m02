Place your documentation and notes files here.

For each dataset, create a notes file following the naming convention:
- `notes-1d-data.md`
- `notes-2d-data.md`
- `notes-1d-multi-method.md`

Each notes file should document:
- Your understanding of the dataset
- Key observations about the data
- Your visualization strategy and design decisions
- How you used AI tools (if applicable) and your own understanding of the process

When you want to include a figure, reference it from the `figs/` folder using relative paths, e.g.:
```markdown
![Figure description](../figs/fig-1d-data.png)
```

See [GitHub markdown guide](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax) for formatting syntax.
